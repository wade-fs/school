台灣高中教師 App 設計建議

# 核心設計原則

台灣高中教師最大的痛點是「角色疊加」：同一個人可能同時是導師、科任教師、還身兼行政職。App 應以角色為軸心，讓每位教師只看到跟自己職責相關的功能，避免介面過度複雜。

# 五大角色的關鍵需求

## 導師

   最迫切需要的是出缺席管理（點名自動化 + 家長通知）以及與家長的聯繫工具。每天重複的點名流程若能縮短到 30 秒內，導師的接受度會大幅提升。

- 出缺席管理
	- 核心需求
	- 一鍵點名（掃碼 / 人臉）
	- 即時推送家長通知
	- 出缺席統計與趨勢圖
	- 遲到 / 請假 審核流程

- 家校聯繫
	- 核心需求
	- 班級群組廣播訊息
	- 個別家長私訊
	- 重要通知已讀回執
	- 家長會排程與簽到

- 班級事務
	- 重要功能
	- 學生基本資料快查
	- 日記、週記批閱
	- 榮譽 / 警告記錄
	- 班費收支紀錄

## 科任教師

   圍繞 108 課綱需求，最需要教材管理（素養導向教案模板）與多元評量記錄。AI 輔助批改是近年最受期待的功能。

- 教材管理
	- 核心需求
	- 教案模板庫 (依 108 課綱)
	- 教材版本控制
	- 跨班複用 / 差異化調整
	- 資源上傳（PDF / 影片 / 簡報）

- 作業與評量
	- 核心需求
	- 線上作業指派與繳交
	- AI 輔助批改初稿
	- 成績登錄與多元評量
	- 題庫管理與隨機出卷

- 學習分析
	- 重要功能
	- 班級成績分佈雷達圖
	- 個別學生學習歷程
	- 弱點診斷與補救建議
	- 跨學期進步追蹤


## 行政教師（教務、學務、總務兼任）

   最需要公文流程數位化與跨部門表單整合，目前很多學校仍靠 Line 群組 + 紙本，效率很低。

- 行事曆與公文
	- 核心需求
	- 校務行事曆同步
	- 公文簽核數位化
	- 會議召集與紀錄
	- 表單申請一站式管理

- 統計報表
	- 重要功能
	- 出缺席統計報告
	- 成績分析彙整
	- 活動參與率報表
	- 客製化匯出（Word / Excel）

- 場地與設備
	- 加分功能
	- 教室 / 場館借用申請
	- 設備庫存盤點
	- 維修申報流程
	- 借還提醒通知

## 輔導教師

   最特殊，資料敏感性極高，個案紀錄必須加密、權限嚴格。心理健康的班級預警功能（定期情緒自評）在後 COVID 時代是強需求。

- 個案管理
	- 核心需求
	- 學生輔導紀錄（加密儲存）
	- 預警機制觸發提醒
	- 轉介流程記錄
	- 跨師合作備忘

- 心理健康評估
	- 核心需求
	- 標準化量表線上施測
	- 全班情緒溫度計週測
	- 高風險名單即時推送
	- 資源轉介連結庫

- 約談排程
	- 重要功能
	- 輔導晤談時段管理
	- 學生自助預約入口
	- 提醒與不赴約紀錄
	- 家長同意書電子版

- 輔導量表與調查模板 (新增需求)
	- 設計各類測驗模板供教師選用：
	- 學生心情溫度計 (每日/每週監測)
	- 生涯規劃與職涯興趣量表
	- 性別平等調查
	- 校園人際關係/愛情調查
	- 心理健康評估工具

## 科主任

   需要的是課程規劃與備課協作空間，尤其 108 課綱後的選修課開設邏輯複雜，視覺化課程地圖是剛需。

- 課程規劃
	- 核心需求
	- 108 課綱學分結構檢核
	- 選修課程開課管理
	- 跨科協作備課空間
	- 課程地圖視覺化

- 教師督導
	- 重要功能
	- 教師教學觀察紀錄
	- 同儕共備會議追蹤
	- 新進教師輔導日誌
	- 專業成長時數統計

- 成效追蹤
	- 加分功能
	- 學科會考 / 學測歷年趨勢
	- 跨班教學成效對比
	- 課程滿意度調查
	- 外部評鑑自評資料

# 值得特別強調的設計考量

- 與現有系統整合

   台灣高中大多使用 SchoolBO（校務行政系統）或各縣市平台，App 應能串接這些系統，而非要求教師重複輸入。

- 離線可用性

   部分山區 / 偏鄉學校網路不穩，出缺席等核心功能應支援離線記錄、上線後同步。

- 通知疲勞問題

   教師通常同時加入數十個 Line 群組，新 App 的通知策略要非常謹慎，否則很快就會被靜音。

- 資料主權

   輔導、學生成績等敏感資料，應明確說明存在台灣境內的伺服器，這對學校採購決策非常重要。
